Harbor Python 批量上传 ACR 操作说明

适用于把 source code/harbor/python/ 下的 python-XXX 任务批量构建成 Docker 镜像并推送到阿里云 ACR。上传动作只在 ECS 上执行，本地 DSW 不执行 build/push。

本地数据目录
/mnt/data/zpy/sci-swe/source code/harbor/python/
ECS 目标目录
/data/harbor-python/
ACR 公网推送地址
agent-rl-acr-registry.cn-beijing.cr.aliyuncs.com
ACR VPC 拉取地址
agent-rl-acr-registry-vpc.cn-beijing.cr.aliyuncs.com
Namespace
sci-swe-python
批量脚本
build_push_all_harbor.sh

0. 当前状态

你已经成功上传了 python-001 和 python-002。当前本地 harbor/python 下有 197 个 python-XXX 目录，缺少 python-003、python-076、python-090。批量脚本只扫描并上传实际存在的目录。

建议正式上传时使用 --skip 001,002，避免重复上传已经完成的两个镜像。

1. 在 DSW 上同步数据到 ECS

这一步只复制文件，不构建镜像。密码或认证方式按你的 ECS 配置执行。

rsync -avz --progress \
  --exclude 'build-logs/' \
  "/mnt/data/zpy/sci-swe/source code/harbor/python/" \
  root@47.236.251.245:/data/harbor-python/

2. 登录 ECS 并准备 ACR

下面所有 Docker 命令都在 ECS 上执行。

ssh root@47.236.251.245
cd /data/harbor-python
chmod +x build_push_all_harbor.sh

确认 namespace 存在并开启自动创建仓库：

aliyun cr CreateNamespace \
  --InstanceId cri-h0hcfm1j42x9wiw6 \
  --NamespaceName sci-swe-python \
  --region cn-beijing || true

aliyun cr UpdateNamespace \
  --InstanceId cri-h0hcfm1j42x9wiw6 \
  --NamespaceName sci-swe-python \
  --AutoCreateRepo true \
  --DefaultRepoType PUBLIC \
  --region cn-beijing

登录 ACR。可以使用你已有的 docker login 方式，或按原 ACR 指南获取 token 后登录：

docker login agent-rl-acr-registry.cn-beijing.cr.aliyuncs.com
如果没有登录或 namespace 未开启 AutoCreateRepo，docker push 通常会报 denied。

3. 先做 dry-run

dry-run 只打印计划执行的 Docker 命令，不会构建、不会推送。

bash /data/harbor-python/build_push_all_harbor.sh \
  --tasks-dir /data/harbor-python \
  --skip 001,002 \
  --dry-run

如果只想测试少数任务：

bash /data/harbor-python/build_push_all_harbor.sh \
  --tasks-dir /data/harbor-python \
  --only 004,045,200 \
  --dry-run

4. 正式批量构建并推送

推荐用 tmux，避免 SSH 断开导致任务中断。

tmux new -s harbor-python-upload

bash /data/harbor-python/build_push_all_harbor.sh \
  --tasks-dir /data/harbor-python \
  --skip 001,002

断开 tmux：按 Ctrl+B，再按 D。重新连接：

tmux attach -t harbor-python-upload
脚本会把每个镜像推送到：
agent-rl-acr-registry.cn-beijing.cr.aliyuncs.com/sci-swe-python/python-XXX:latest

5. 查看日志和失败任务

总日志和结果清单都在 /data/harbor-python/build-logs/。

tail -f /data/harbor-python/build-logs/summary.log

cat /data/harbor-python/build-logs/success.txt
cat /data/harbor-python/build-logs/failed.txt
cat /data/harbor-python/build-logs/skipped.txt

每个任务还有单独日志，例如：

tail -80 /data/harbor-python/build-logs/python-045.log

6. 重试失败任务

如果只有少数任务失败，用 --only 重试，不需要重跑全部。

bash /data/harbor-python/build_push_all_harbor.sh \
  --tasks-dir /data/harbor-python \
  --only 045,068,200

如果怀疑 Docker 缓存导致问题：

bash /data/harbor-python/build_push_all_harbor.sh \
  --tasks-dir /data/harbor-python \
  --only 045 \
  --no-cache

7. 验证 ACR 镜像

列出 namespace 下的仓库：

aliyun cr ListRepository \
  --InstanceId cri-h0hcfm1j42x9wiw6 \
  --RepoNamespaceName sci-swe-python \
  --PageSize 100 \
  --region cn-beijing

拉取并验证一个镜像：

docker pull agent-rl-acr-registry.cn-beijing.cr.aliyuncs.com/sci-swe-python/python-004:latest

docker run --rm \
  agent-rl-acr-registry.cn-beijing.cr.aliyuncs.com/sci-swe-python/python-004:latest \
  /bin/bash -lc 'PYTHONPATH=/app/workspace bash /tests/test.sh'
ACS 沙箱里应使用 VPC 地址：
agent-rl-acr-registry-vpc.cn-beijing.cr.aliyuncs.com/sci-swe-python/python-XXX:latest

8. 清理 ECS 磁盘

脚本默认每个任务推送后会清理本地镜像。必要时可以额外清理 Docker 缓存：

docker image prune -a -f
docker builder prune -f
df -h /
不要在上传任务运行中执行全量 prune，避免影响正在构建的镜像。
