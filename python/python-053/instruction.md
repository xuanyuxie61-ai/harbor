# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：
该项目涉及某种动力学模型（如RDO模型）的数值模拟。缺失的代码分布在三个逻辑位置：
1. 数值积分调用：此处需要实现常微分方程的数值求解。请调用项目已有的四阶龙格-库塔积分函数，传入微分方程函数、初始状态变量、起始时间（值为0.0）、结束时间（以年为单位的变量）以及总步数变量，并返回计算得到的时间序列和状态序列。
2. 结果日志输出：此处需要输出积分完成后的状态信息。请打印RDO积分完成的提示，包含最终时间（保留一位小数，单位为年）和总步数；同时打印最终状态，提取状态序列最后一行的前两个分量（分别对应h_W和T_E），并保留四位小数。
3. 数据校验：此处需要对输入的轨迹数据进行维度验证。判断轨迹数据的维度属性是否为2，若不是，则抛出值错误异常，并提示轨迹必须是二维数组。
