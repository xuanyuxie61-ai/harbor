# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：

本项目实现了一个双物种竞争与传染病交叉传播的SIR动力学模型。项目中需要补全三个核心代码片段：

片段一：基于空间场的双物种竞争-传染病模型微分方程导数计算。该部分需要实现带有种间竞争系数的逻辑斯谛增长方程。为了保证数值稳定性，需对环境容量设置下限以防除零，并对总种群进行上下限截断以防止无界增长。接着计算包含种间交叉传播的感染力，同样需要对易感者、感染者种群以及感染力本身进行上限截断保护。此外，当感染者数量超过给定阈值（该阈值需从参数字典获取，键名自拟，默认值为0.1）时，需要调用项目中已有的自催化感染率函数（传入截断后的易感者和感染者，以及两个固定为0.05的小参数）来实现Selkov型自催化增强机制。最后，综合计算两个物种的易感者、感染者、康复者仓室的导数，对所有导数结果进行截断保护，并将最终结果按特定顺序堆叠返回。

片段二：基础标量模式下的双物种竞争-传染病模型导数计算。该部分同样需要实现包含种间竞争系数的逻辑斯谛增长方程，以及包含种间交叉传播的感染力计算，但无需任何数值截断保护或自催化增强。直接按照标准SIR模型结构计算各仓室的导数，并将结果按顺序以数组形式返回。

片段三：基本再生数的评估。该部分需要基于当前易感者数量、种间传播率以及各物种的恢复率与死亡率，实现多物种交叉传播下的基本再生数计算。最终返回一个字典，包含两个物种基本再生数在整个空间上的均值和最大值统计量。

注意：项目中已有的其他函数（如自催化感染率函数和基础数学库等）默认已正确实现，可直接调用，不需要重新定义。
