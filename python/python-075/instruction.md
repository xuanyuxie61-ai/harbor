# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：

片段1：本片段主要实现基于简化4组分3步反应机制的化学反应速率、组分生成率及温度源项的计算。首先，需要定义3个反应的阿伦尼乌斯参数：指前因子（分别设为10的5次方、4次方和3次方量级）、温度指数（均为0）以及活化能（分别设为8万、6万和10万焦耳每摩尔）。接着，需要利用阿伦尼乌斯公式计算各个反应的正向反应速率常数，并根据各反应的反应物浓度计算各个反应的速率。然后，通过构建化学计量矩阵（注意矩阵维度与转置操作）与反应速率向量的乘积，求得各组分的生成率。最后，计算温度源项，此处需要调用已有的焓函数和混合比热函数，根据能量方程计算热释放引起的温度变化。如果输入是标量，则返回单点结果，否则返回全场结果。

片段2：本片段实现H2-空气系统中化学计量混合分数的计算。首先定义H2和O2的分子量（分别约为2.01588e-3和31.9988e-3 kg/mol），并据此计算质量化学计量比。接着，从输入的燃料和氧化剂字典中提取燃料中H2的质量分数（默认为1.0）和氧化剂中O2的质量分数（默认为0.233）。最后，基于质量化学计量比和质量分数，利用化学计量混合分数公式计算并返回该值。

片段3：本片段实现基于混合分数和反应进度变量到多组分质量分数场的映射与初始化。建立一个与网格维度匹配的零数组用于存储各组分场。根据线性近似关系映射各组分：H2与混合分数及未反应部分相关，O2与未混合部分相关，H2O与混合分数及反应进度相关，N2则由1减去前三个组分之和得到。映射后，需要将所有质量分数截断在0到1之间，并在网格的每个点上对这4个组分的质量分数进行归一化处理，以确保总和严格为1。
