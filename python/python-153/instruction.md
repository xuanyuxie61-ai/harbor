# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：

首先，缺失了一个用于一维对流方程单步时间推进的函数。该函数接收一维数组、速度系数以及空间和时间步长作为参数。函数内部需验证输入数组必须为一维，步长必须为正数，且数组长度至少为3。在周期性边界条件下，此处需要实现一维对流方程的FTCS（前向时间中心空间）离散格式，通过当前点及其相邻两点的值计算下一时间步的值，最后返回更新后的数组。

其次，缺失了一个用于计算FTCS格式von Neumann放大因子绝对值的函数。该函数接收速度系数、时间步长、空间步长和波数值数组。验证步长为正后，需计算CFL数，此处需要按照FTCS格式的von Neumann稳定性分析公式计算复数放大因子，并返回其绝对值数组。

最后，缺失了一段演示与验证代码。首先初始化一个长度为64的一维零数组，生成0到1范围内的等间距空间坐标，并在0.4到0.6的区间内，利用一个两端为零、中间平滑凸起的多项式设置初始条件。接着，调用上述单步推进函数，设定速度系数为1.0，时间步长为0.001，空间步长为1/63，打印推进前后的向量范数变化。随后，在0到pi的范围内生成50个波数点，调用放大因子计算函数，其中空间步长设为0.01，其余参数不变，计算并打印放大因子的最小值与最大值范围，最后通过判断最大放大因子是否大于1来验证FTCS的无条件不稳定性。
