# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：

片段1对应于单步化学反应速率的计算函数。该函数需要基于阿伦尼乌斯方程计算基础反应速率，其中指数部分需要进行截断处理以防数值溢出。同时，需引入双限温度因子来确保点火门槛和绝热上限的限制：下限点火温度设为700.0，上限为化学当量绝热温度。该温度因子需要进行归一化处理使峰值约为1，并截断在0到1之间。此外，如果混合分数变量存在，需要根据其与化学当量混合分数的偏差，施加基于高斯分布的空间限制因子（标准差设为0.10），并将该因子截断在极小正值到1之间。最终的反应速率为基础速率、温度因子和空间因子的乘积。

片段2对应于热源项的计算函数。需要先基于典型条件（典型温度1400.0，化学当量下燃料质量分数0.03与氧化剂质量分数0.18）调用已有的单步反应速率函数，获取峰值反应速率。接着结合热释放、参考密度（0.18）与定压比热容（1200.0）计算最大源项，并将其截断在100.0到2000000.0的合理范围内。随后，计算基于高斯分布的空间分布因子（标准差为0.06），以及截断在0到1之间的温度自限因子（用以保证温度接近绝热温度时源项趋于0）。最终的热源项由最大源项、空间因子与温度自限因子相乘得到。

片段3是对反应源项进行线性化处理的逻辑。需要调用已有的单步反应速率函数来计算反应速率，在调用时需根据当前的组分类型（燃料或氧化剂）正确传入质量分数参数：若为对应类型则传入当前质量分数，否则传入固定值0.5。最后，计算简化的源项，其值等于负的反应速率除以当前密度。
