# 任务：修复挖空代码（Multi-Hole Benchmark）

## 目标
你面对的是一组Python 的科研代码。代码中有多处被"挖空"（函数体、关键逻辑、边界条件等被删除或替换为占位符），导致代码无法正确运行或输出错误结果。

## 工作目录
代码仓库位于 `/app` 目录下。

## 要求
你现在在一个项目中，里面缺失了一部分的代码，请你找到缺失的位置并补全。代码的简介如下：

第一个缺失片段是一个计算Lennard-Jones（LJ）作用力大小的函数。该函数接收粒子间距离、能量参数（默认值为1.0）和尺寸参数（默认值为1.0）作为输入。在此处需要实现LJ势能对距离的解析导数方程，算法核心在于计算距离比例的6次方和12次方项，并注意需对输入距离进行合法性校验，确保距离必须大于零。

第二个缺失片段是分子动力学模拟中执行Velocity Verlet时间步的方法。该方法接收一个布尔参数以决定是否应用恒温器（默认值为真）。算法流程需严格遵循Velocity Verlet积分规则：首先根据当前速度和加速度更新位置，随后应用周期性边界条件；接着保存旧加速度，通过调用已有的方法计算新的力和加速度；利用新旧加速度的平均值更新速度；若启用恒温器，则调用已有的Berendsen恒温器方法并重新计算动能和温度；最后调用已有方法计算压强和总能量，并返回势能、动能、温度、压强和总能量这五个热力学量。

第三个缺失片段是一个从能量时间序列计算定容热容的函数。该函数接收能量序列、系统温度、粒子数、空间维度（默认值为2）和玻尔兹曼常数（默认值为1.0）作为输入。此处需要实现基于统计力学涨落理论的定容热容计算方程，即通过能量的方差与温度平方的关系来求解。同时还需要计算并返回基于杜隆-珀替定律的热容理论极限值。算法需注意处理能量序列长度不足以及温度极低可能导致除零的边缘情况。
